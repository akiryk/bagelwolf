# SEO & Accessibility Guide

This document describes the SEO and accessibility implementation for the Bagelwolf website.

---

## Overview

The site implements:

- **Meta tags** — title, description for each page
- **Open Graph** — social sharing previews (Facebook, Instagram, Threads)
- **Twitter Cards** — Twitter/X sharing previews
- **Sitemap** — auto-generated XML sitemap
- **robots.txt** — search engine crawling instructions
- **Semantic HTML** — proper heading hierarchy, landmarks
- **Accessibility** — alt text, keyboard navigation, contrast

---

## SEO Component

**File:** `src/components/SEO.astro`

This reusable component generates all meta tags. It's automatically included via `BaseLayout.astro`.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | string | required | Page title (shows in browser tab and search results) |
| `description` | string | required | Meta description (150-160 characters recommended) |
| `image` | string | `/images/bagel-wolf-logo.png` | Open Graph image URL |
| `type` | `'website'` \| `'article'` | `'website'` | OG content type |
| `noindex` | boolean | `false` | Set `true` to hide page from search engines |

### Usage

When creating a new page, pass SEO props to `BaseLayout`:

```astro
---
import BaseLayout from '../layouts/BaseLayout.astro';
---

<BaseLayout
  title="Page Title | Bagelwolf Bakery"
  description="A compelling description of this page in 150-160 characters. Include keywords naturally."
  image="/images/custom-og-image.png"
>
  <!-- Page content -->
</BaseLayout>
```

### Default Values

If you only pass `title`, the layout uses these defaults:
- **description:** Generic Bagelwolf description
- **image:** Logo image

## Sitemap

**Generated by:** `@astrojs/sitemap` integration

**Configuration:** `astro.config.mjs`

The sitemap is automatically generated at build time and includes all pages.

**URLs:**
- Index: `https://bagelwolfbakery.com/sitemap-index.xml`
- Pages: `https://bagelwolfbakery.com/sitemap-0.xml`

### Excluding Pages

To exclude a page from the sitemap, you can configure the integration in `astro.config.mjs`:

```js
sitemap({
  filter: (page) => !page.includes('/private/')
})
```

## robots.txt

**File:** `public/robots.txt`

Current configuration allows all crawlers to access all pages:

```
User-agent: *
Allow: /

Sitemap: https://bagelwolfbakery.com/sitemap-index.xml
```

## Open Graph Images

For best social sharing previews:

- **Recommended size:** 1200×630 pixels
- **Format:** PNG or JPG
- **Location:** `/public/images/`

Each page can specify a custom OG image via the `image` prop.

2. Use browser dev tools → Elements → search for "og:"

After deploy, verify at:
- `https://bagelwolfbakery.com/robots.txt`

## Accessibility Checklist

The site follows these accessibility practices:

- [x] All images have `alt` text
- [x] Heading hierarchy: H1 → H2 → H3 (no skipping levels)
- [x] Semantic HTML landmarks (`<header>`, `<main>`, `<footer>`, `<nav>`)
- [x] Focus styles on interactive elements
- [x] Sufficient color contrast (Tailwind defaults meet WCAG AA)
- [x] `lang="en"` on `<html>` element
- [x] Keyboard-navigable links and buttons

### Testing Accessibility

Run Lighthouse audit in Chrome DevTools:
1. Open DevTools (F12)
2. Go to Lighthouse tab
3. Select "Accessibility" category
4. Run audit

Target score: **≥ 90**
